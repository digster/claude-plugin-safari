# 2026-03-18 — Green Dot Badge Notification + Smaller Dot Refactor

## Summary

Added toolbar icon badge notifications, then replaced the oversized `setBadgeText` API with canvas-based icon compositing for a smaller, subtler green dot (~15% of icon size) in the top-right corner.

## Changes

### Session 1: Badge Notification (initial implementation)
- Added `urlToTabId` Map, `setBadge`/`clearBadge`/`clearBadgeForUrl` using badge text API
- Popup sends `clearBadge` on showing complete/error results
- Tab lifecycle cleanup listeners

### Session 2: Smaller Dot via Canvas Compositing

#### `extension/background.js`
- Replaced `setBadgeText`/`setBadgeBackgroundColor` with `setIcon({ imageData })` using canvas compositing
- Added `createDotIcon(size, color)` — loads icon PNG, draws small dot overlay, caches `ImageData` by `"size-color"` key
- `setBadge` is now async — composites 16px and 32px icons in parallel via `Promise.all`
- `clearBadge`/`clearBadgeForUrl` now async with `await`
- Added `clearBadgeForTab(tabId)` — clears badge and removes all URL→tab map entries for that tab
- `clearBadge` message handler now supports both `tabId` and `url` params
- `tabs.onUpdated` listener is now async

#### `extension/popup/popup.js`
- Sends single `clearBadge` with `tabId` unconditionally on init (opening popup = user is looking at tab)
- Removed per-branch `clearBadge` calls from complete/error handlers (init, click, poll)

#### `tests/background.test.js`
- Replaced `browser.action` mock: `setIcon` + `_icons` tracking instead of `setBadgeText` + `_badges`
- Added `browser.runtime.getURL` mock

- New tests: `clearBadgeForTab`, `clearBadge` by tabId message, `setIcon` API failure resilience

### Session 3: Fix Spinner Hang — Non-blocking Badge + Image Load Robustness

Commit `c11862c` introduced two critical bugs: (1) `setBadge()` became async and was `await`ed in `runClaude()`, blocking result delivery if image loading hangs; (2) `createDotIcon()` sets `img.src` before attaching `onload`/`onerror`, risking a race condition.

#### `extension/background.js`
- **Fire-and-forget badges in `runClaude()`** — removed `await` from all `setBadge()` and `clearBadgeForUrl()` calls inside `runClaude()` (lines 253, 270, 287, 300). Badge rendering is cosmetic and must never block result delivery.
- **Fixed `createDotIcon()` image loading** — attach `onload`/`onerror` handlers BEFORE setting `img.src` to avoid race conditions. Added `img.complete` check to skip waiting if image is already cached. Added 3-second timeout via `Promise.race` as safety net against hanging loads.

#### `tests/background.test.js`
- Updated `Image` mock: constructor initializes `_onload`/`_onerror`/`_complete`, `onload` fires from `src` setter (matching handler-before-src pattern), added `complete` getter
- Added microtask flushes (`await new Promise(r => setTimeout(r, 0))`) after fire-and-forget badge tests

### Session 4: Cache Explorer + Remove LRU Eviction Cap

Added a full-page Cache Explorer (sidebar+detail view) for browsing, viewing, copying, and deleting cached results. Removed the 25-file LRU eviction cap since native disk has no meaningful size constraint — users manage cache via the explorer instead.

- Added `deleteCachedResult` action — takes `url` param, reuses `resultFileURL(for:)` SHA-256 lookup, deletes single file
- Removed `evictOldResults()` method entirely and its call from `handleStoreResult()`
- Updated file header comments to list new actions

#### `extension/background.js`
