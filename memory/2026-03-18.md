# 2026-03-18 — Green Dot Badge Notification

## Summary

Added toolbar icon badge notifications so users can tell when a fetch completed without opening the popup.

## Changes

### `extension/background.js`
- Added in-memory `urlToTabId` Map to track which tab initiated each fetch
- Added `setBadge(tabId, status)` — sets green dot (`#34C759`) for complete, red dot (`#FF3B30`) for error, clears on null
- Added `clearBadge(tabId)` and `clearBadgeForUrl(url)` helpers
- Modified `runClaude(url, tabId)` — stores mapping, sets green badge on complete, red on error, clears on cancel
- Modified `cancelClaude(url)` — clears badge immediately
- Added `clearBadge` message action in the message handler
- Added `browser.tabs.onUpdated` listener — clears badge when tab navigates away
- Added `browser.tabs.onRemoved` listener — cleans up map entries on tab close
- Exported `setBadge`, `clearBadge`, `clearBadgeForUrl`, `urlToTabId` for testing

### `extension/popup/popup.js`
- Added `currentTabId` variable, captured from `tabs[0].id`
- Sends `tabId` with `runClaude` message
- Sends fire-and-forget `clearBadge` messages when displaying complete/error results (init, click handler, poll)

### `tests/background.test.js`
- Added `browser.action` mock with `_badges` tracking object
- Added `browser.tabs.onUpdated`/`onRemoved` listener mocks
- 14 new test cases: setBadge colors, clearBadge, clearBadgeForUrl, badge on runClaude complete/error, cancelClaude badge clear, clearBadge message action, tab navigation/close cleanup, badge API failure resilience, runClaude without tabId

### `tests/popup.test.js`
- Added `clearBadgeMessages` and `runClaudeMessages` tracking arrays
- Added `id: 123` to mock tab query results
- 5 new test cases: clearBadge on init (complete/error), no clearBadge when running, tabId in runClaude, poll clearBadge

### `ARCHITECTURE.md`
- Updated component descriptions and added badge notification design decision
